Siddhi Family Guthi — Data Maintenance NoteAll names, lists, and dates live in two files. You never touch HTML or CSS.Open the file, edit between the brackets, save, and commit to GitHub.---1. Thaipa / Seven Thakalis (elders)File: data.js → the `ELDERS` list.Each elder is one line:    { name:"Purna Siddhi", birthYear:1938, lineage:"Thaipa — senior-most", note:"Head of the seven Thakalis." },name — full namebirthYear — year of birth in English (AD); age is calculated automaticallylineage — short relation line (optional, use "" if none)note — one extra line shown in italics (optional)Keep the senior-most first. Order in the file = order on the page.To add an elder, copy a line and edit it. To remove one, delete the line.---2. Executive CommitteeFile: data.js → the `COMMITTEE` block.term — the committee's term shown at the top (e.g. "2082–2084 B.S.")Each member is one line:{ name:"Surendra Siddhi Bajracharya", position:"Chairman", period:"2082–2084", phone:"", email:"" },position — Chairman, Secretary, Treasurer, Member, etc.phone / email — optional; leave as "" if none. They become clickable when filled.Keep the Chairman first. Add or remove member lines freely.---3. Gatherings & Quick Event ListFile: app.js → the `EVENTS` list. (Both the "Upcoming events" panel and the"Quick list" are filled from this one list — you only edit it once.)Each event is one line:    { date:"2026-07-24", name:"Annual Guthi Puja", meta:"Guthi Ghar · 9:00 AM", tag:"Ritual" },date — English date only, format YYYY-MM-DD. The Nepali (BS) date appearsautomatically; you never type the Nepali date.name — event titlemeta — place and time (optional)tag — short label: Ritual, Festival, Meeting, etc.Events sort by date on their own. Delete past events when you like.---4. Fortnightly prey turnFile: app.js → the `PREY` block.startDate — the date the FIRST member's turn began (YYYY-MM-DD). Set once.fortnightDays — leave as 14.members — the fixed list, in order. One name per line:{ name:"Member Name" },Rules:Add a NEW member to the END of the list.When someone passes away, mark them — do NOT delete:{ name:"Late Member Name", deceased:true },They keep their place but are skipped, and the turn moves to the next living member.The current turn is calculated automatically from the start date. You neverset "whose turn it is" by hand.---Leadership trio on the front page (Thaipa / Chairman / Pala)File: app.js → the `LEADERSHIP` block. Three lines — edit the names here whena role changes. The Pala line has `pala:true`, which keeps its gold highlight.---Quick remindersEdit only inside the quotes and brackets. Keep the commas and the line format.After editing, save the file and push to GitHub. Changes go live on refresh.Dates: always enter English YYYY-MM-DD. Nepali dates are produced for you.BS calendar is accurate through 2033 AD (BS 2090).If something looks broken, you likely removed a comma or a bracket — undo and retry.
